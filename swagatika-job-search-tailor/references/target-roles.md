## Avoid or Deprioritize

- roles centered on team management
- pure data science / machine learning roles
- software engineering roles
- roles requiring deep SAS-only backgrounds unless the rest of the match is unusually strong
- roles that need direct clinical credentialing

## Fit Heuristics

Strong fit if the role asks for:
- healthcare or payer data
- SQL-heavy analytics
- ETL validation or reconciliation
- data quality / governance
- dashboarding or reporting
- cross-functional stakeholder work

## Technology Mapping Rule

- Do not add unsupported tools or technologies to the resume as if they were used directly.
- If the posting asks for a neighboring tool, map to adjacent real experience:
- `Tableau` or another BI tool can be positioned next to real `Power BI` and reporting experience only if clearly marked through wording that remains truthful.
- `Snowflake`, `Databricks`, `Python`, or cloud tools can be emphasized only to the level already supported by the base resume.
- `Prompt engineering`, `ChatGPT`, `Copilot`, and AI-assisted analytics can be positioned as adjacent learning or productivity support only when the resume supports it; do not claim professional prompt-engineering ownership unless explicitly present in the source profile.
- Unsupported requirements should be tracked in `gaps` and `notes`, not fabricated into experience bullets.

## Required Metadata Per Role

- `source`
- `employment_type`
- `work_mode`
- `location`
- `job_url`
- `fit_score`

Use `Unknown` only when the posting genuinely does not expose the field or the current search result does not provide it.

Medium fit if the role is broader data analysis but still matches:
- healthcare domain
- SQL
- large-scale data validation
- AI-assisted analysis or prompt-assisted reporting layered onto SQL, healthcare analytics, data quality, or BI/reporting work
- dashboards, reporting, Excel/BI tools, stakeholder analytics, data integrity, or cross-functional business analysis, even outside healthcare

Weak fit if the role is dominated by:
- hands-on data science modeling
- people management
- pure prompt engineering with no SQL/data-analysis responsibilities
- unrelated domains with no transferable overlap
